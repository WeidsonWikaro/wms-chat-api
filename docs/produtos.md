# Produtos

## Para que serve

Cada artigo físico que circula no armazém deve existir como **produto** no sistema, com nome e **código de barras único**. Todo o stock, picking, recebimento e contagens referenciam estes produtos.

## Regras importantes

1. **Código de barras único** — Não pode haver dois produtos com o mesmo código. Na recepção ou no picking, o código de barras é a forma mais rápida de garantir que está mexendo no artigo certo.
2. **Consulta por código** — Para identificar um artigo desconhecido no caixote, use sempre o leitor ou a pesquisa por código de barras **exato**.
3. **Pesquisa por texto** — Quando não tem o código, pode procurar por palavras do nome ou da descrição; confirme sempre visualmente o artigo antes de movimentar stock em grande quantidade.
4. **Alterações** — Se mudar o código de barras de um produto já com stock, coordene com a equipe: etiquetas antigas deixam de bater certo com o sistema.

## Boas práticas

- Mantenha **nome** e **descrição** úteis para quem está no chão (tamanho, cor, embalagem).
- Alinhe com o **ERP ou sistema de vendas** para o mesmo SKU não ter nomes diferentes em sistemas distintos.

## Quem usa mais este módulo

- **Recepção** — Conferir se o artigo que chegou existe e está correto.
- **Picking** — Confirmar o produto da linha (especialmente quando há substitutos visualmente parecidos).
- **Contagem** — Garantir que está contando o SKU certo no local certo.
